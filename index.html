Project Work

Inventory Lookup and Reporting Application

During my time as an Inventory Administrative Support staff member at PT. Euroasiatic Jaya, inventory records were managed through manual processes. The company used a simple desktop program that worked mainly as a digital stock card (kartu stok).

The existing inventory system had very limited functions and only allowed basic stock lookup and simple reports. To obtain more detailed information, sort data, or create customised reports, the data often had to be checked and made manually using spreadsheets. This process was time-consuming, made it difficult to obtain accurate inventory information, and increased the risk of errors.

The goal of this project was to develop a simple desktop application to make it easier to retrieve inventory information and improve the inventory reporting process. I independently designed and developed the application using Visual Basic and SQL, based on the existing inventory data used by the organisation.

The application was designed based on the existing system but developed using more modern tools and a database, allowing additional and improved features. It allows users to search inventory records by item name, supplier, customer, and other categories. Users can also view stock movement data and generate reports through a user-friendly interface. The system organised existing inventory data into a more structured format by combining the original database with manual records previously maintained in spreadsheets (stock cards). This made it possible to retrieve stock information and prepare inventory reports automatically using a single application.
